title: "25w36a Changelog"
date: 2025-09-25

<p>Snapshot 25w36a is out and it includes mannequins, a new technical entity that is spawned with commands and can be used as an NPC. Conflicting key binds will now function together allowing you to bind a key to multiple actions. And as always, this snapshot also includes technical tweaks and bug fixes.<blockquote><blockquote><p><strong>Developer's Note</strong>: <em>This snapshot brings a large change to how we collect and draw block entities and particles. We don't anticipate any change of behaviors so please keep an eye out and report anything suspicious as a bug, thank you!</em></blockquote></blockquote><h1>New Features</h1><ul><li>World Borders can now be set per dimension</ul><h2>End Light Flashes</h2><ul><li>While the Ender Dragon fog effect is active the flash source in the sky is not visible and the brightness of the light affecting the world is reduced</ul><h2>Accessibility</h2><ul><li>Multiple key binds that have the same key assigned to them can now work together, while before only one would work<ul><li>For example, you can now bind <code>D</code> to <code>Jump</code> and <code>Strafe Left</code> to jump and strafe left at the same time<li>The red warning for assigning the same key to multiple key binds is changed to yellow</ul><li>The spectator hotbar menu key can now be rebound<ul><li>It is part of a new <code>Spectator</code> category in the Key Binds screen, alongside the pre-existing <code>Highlight Players</code> key bind</ul></ul><h1>Changes</h1><ul><li>Formatting codes are now stripped from Code of Conduct files</ul><h2>World Border</h2><ul><li>The World Border is now dimension-specific<ul><li>This means that each dimension can have its own World Border size, position, etc.</ul></ul><h2>Minor Tweaks to Blocks, Items and Entities</h2><ul><li>Copper Torch and Copper Lantern colors are adjusted to make them more discernible to color blind players</ul><h2>UI</h2><ul><li>Double-clicking a word in edit box now selects it</ul><h2>Block States</h2><ul><li>Powered Shelves can now only connect to other Powered Shelves facing the exact same direction</ul><h2>Performance Improvements</h2><ul><li>Particles outside of players view are not rendered anymore, which provides a performance improvement in some cases</ul><h1>Technical Changes</h1><ul><li>The Data Pack version is now 86.0</ul><h1>Data Pack Version 86.0</h1><ul><li>Added Mannequin entities<li>The World Border is now dimension-specific</ul><h2>Mannequins</h2><p>Added a new type of technical entity called <code>minecraft:mannequin</code> which can only be spawned with summon commands.<ul><li>A Mannequin is a Player Avatar without a connected Player<li>Mannequins always show an "NPC" text where a Player's <code>below_score</code> score would show<li>Mannequins function as Living Entities - they can hold and wear equipment, have attributes &#38 effects, take damage, etc</ul><p>Data Fields:<ul><li><code>profile</code> - Which profile to show on the entity. This is either a Player profile (same format as a <code>minecraft:profile</code> component) or a Mannequin profile with the following fields:<ul><li><code>texture</code> - Namespaced ID of the skin texture to use<ul><li>The skin is specified relative to the <code>textures</code> folder and with a <code>.png</code> suffix<li>e.g. <code>entity/player/wide/steve</code> will use the default wide Steve skin</ul><li><code>cape</code> - Optional namespaced ID of the cape texture to use<ul><li>When specified, uses the same format as the <code>texture</code> field<li>Omitting this field means the Mannequin has no cape</ul><li><code>elytra</code> - Optional namespaced ID of the elytra texture to use<ul><li>When specified, uses the same format as the <code>texture</code> field<li>Omitting this field means the Mannequin uses the cape texture, or if that is also omitted, the default Elytra texture when wearing Elytra</ul><li><code>model</code> - Optional model type, one of <code>wide</code> and <code>slim</code><ul><li>If omitted, <code>wide</code> is used</ul></ul><li><code>hidden_layers</code> - List of outer skin layers to hide<ul><li>Valid entries: <code>cape</code>, <code>jacket</code>, <code>left_sleeve</code>, <code>right_sleeve</code>, <code>left_pants_leg</code>, <code>right_pants_leg</code>, <code>hat</code></ul><li><code>main_hand</code> - Which hand is the main hand of the Mannequn - one of <code>left</code> and <code>right</code></ul><h2>Particles</h2><ul><li><code>dragon_breath</code> now optionally takes a <code>power</code> parameter (float, default <code>1.0</code>), which is multiplied onto its initial velocity, after all randomness<li><code>instant_effect</code> and <code>effect</code> now optionally take <code>power</code> (float, default <code>1.0</code>) and <code>color</code> (rgb, default <code>0xFFFFFF</code>) parameters<li><code>flash</code> now requires a <code>color</code> parameter (argb)</ul><h1>Resource Pack Version 68.0</h1><h3>Shaders</h3><ul><li>The following unused shaders have been
