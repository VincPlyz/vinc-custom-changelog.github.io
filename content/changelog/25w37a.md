<p>A new week, and a new snapshot. This one includes another big batch of bug fixes, copper golem polishing and our beloved friend can now open doors.<p>Happy mining!<h1>Changes</h1><ul><li>OpenGL 3.3 is now required to play the game, up from 3.2. This does not change our posted Minimum Requirements</ul><blockquote><p><strong>Developer’s Note:</strong> <em>We do not expect the update to OpenGL version to affect many people, if any at all. Our posted Minimum Requirements has been OpenGL 4.0 since 2017, and OpenGL 3.3 turned 15 years old recently.</em></blockquote><h2>Minor Tweaks to Blocks, Items and Entities</h2><ul><li>Fixed a bug where breaking a Shelf while another player is interacting with it could lead to visual bugs in the player inventory<li>Copper Golems hitbox has been tweaked<ul><li>The hitbox is now 0.49 blocks in width to be able to pathfind through narrow spaces</ul><li>Copper Golems can now open doors<li>Test Instance Blocks now preserve error markers between world reloads<li>Ambient lighting on entities has been adjusted to be more visually in line with blocks and other parts of the game<ul><li>The underside of flat surfaces will now be lit from that direction, instead of the opposite direction<li>This mainly affects models containing cubes that are either flat or "hollow" such as:<ul><li>Worn armor<li>Worn Elytra<li>The outside layers on player skins<li>The ribcages of Skeletons or Skeleton Horses<li>The feet of Chickens or Frogs<li>The wings of Phantoms or the Ender Dragon<li>The fins on fish<li>And more!</ul></ul></ul><h2>Game Mode Switching</h2><ul><li>Switching from Spectator Mode to Creative Mode no longer causes the player to stop flying<ul><li>Instead, flight now only gets disabled when switching game modes if the player is near the ground<ul><li>A player is considered to be near the ground if there is a collidable surface (i.e. blocks, Happy Ghasts, boats...) within 1 block below them</ul><li>Flight will not get disabled if the player was inside a block or collidable entity<ul><li>This is to prevent players from suddenly falling through the ground</ul></ul></ul><blockquote><p><strong>Developer's Note</strong>: <em>This change is a partial revert of a bugfix from a previous snapshot. Even though the old behavior prior to the fix was unintended, the approach we took to fixing flight didn't quite <strong>land</strong> the way we wanted it to. In response to community feedback, we're trying this new approach as a less disruptive alternative.</em></blockquote><h2>Minecraft Server Management Protocol</h2><ul><li>Clients must authenticate to access the API<ul><li>Clients should supply an <code>Authorization</code> bearer token header with a server specific secret<li>The secret is configured in the server properties file<ul><li><code>management-server-secret=</code><li>The secret should be exactly 40 alphanumeric characters (A-Z, a-z, 0-9)<li>The secret will be automatically generated if the server property is empty<li>Unauthorized requests are rejected with 401 Unauthorized</ul></ul><li>TLS is enabled by default<ul><li>Can be disabled in the server properties file<ul><li><code>management-server-tls-enabled=false</code></ul><li>Requires a keystore file to be set in the server properties file<ul><li><code>management-server-tls-keystore=path/to/keystore.p12</code></ul><li>The keystore file must be in PKCS12 format<li>Keystore password can be set in the following ways, in order of priority<ul><li>Environment variable: <code>MINECRAFT_MANAGEMENT_TLS_KEYSTORE_PASSWORD</code><li>JVM argument: <code>-Dmanagement.tls.keystore.password=</code><li>Server property: <code>management-server-tls-keystore-password=</code></ul></ul></ul><h2>Server Properties</h2><ul><li>Default value of <code>management-server-port</code> was changed from <code>25585</code> to <code>0</code>, assigning an available port each time management server is started<li>Added server property <code>management-server-tls-enabled</code> controlling whether the Minecraft Server Management Protocol uses TLS or not<ul><li>Defaults to <code>true</code></ul><li>Added server property <code>management-server-tls-keystore</code> controlling the path to the keystore file used for TLS<ul><li>Defaults to an empty string<li>A dedicated server won't start when TLS is enabled and no keystore is provided</ul><li>Added server property <code>management-server-tls-keystore-password</code> controlling the password to the keystore file used for TLS<ul><li>Defaults to an empty string</ul></ul><h2>Game Rules</h2><ul><li>Added <code>spawnerBlocksEnabled</code> game rule controlling if Spawner Blocks should be enabled or not<ul><li>Defaults to <code>true</code></ul></ul><h2>Mob Spawning</h2><ul><li>Logic for choosing type of equipment for mobs spawning with equipment has been tweaked to add Copper as a possibility<ul><li>The initial roll to choose a type now chooses between Leather, Copper and Gold<li>The chance to increase an armor tier has increased from 9.5% to 10.87%</ul></ul><h2>Sounds</h2><ul><li>Some sound sliders in the Music &#38 Sound Options now play a preview sound when adjusting the setting outside a world</ul><h2>UI</h2><ul><li>The Resource Pack and Data Pack selection screens now have a search box</ul><h1>Technical Changes</h1><ul><li>The Data Pack version is now 87.0<li>The Resource Pack version is now 69.0<li>Some previously hidden debug features have been exposed to players</ul><h2>Debug features</h2><ul><li>Some debug features that were previously removed during compilation are now preserved and can be used by players<li>Individual values can be enabled with JVM properties:<ul><li>First, a global debug flag must be enabled with <code>-DMC_DEBUG_ENABLED</code> or <code>-DMC_DEBUG_ENABLED=true</code><li>Then, a specific feature can be enabled, for example <code>-DMC_DEBUG_BRAIN</code> or <code>-DMC_DEBUG_BRAIN=true</code><li>To get the list of all available debug properties, add <code>-DMC_DEBUG_PRINT_PROPERTIES</code></ul><li>Warning: this is debug code. Handle with extreme care!<ul><li>This is an advanced feature, meant to help modders - it should not be used for normal play<li>Features not guaranteed to work as intended<li>It might crash your game or corrupt your worlds<li>Individual options might be added, changed or removed without a notice</ul></ul><h1>Data Pack Version 87.0</h1><ul><li>Changed name of game rule <code>enableCommandBlocks</code> to <code>commandBlocksEnabled</code></ul><h1>Resource Pack Version 69.0</h1><h2>Textures</h2><ul><li>Moved the result slot of <code>gui/container/villager.png</code> up by one pixel</ul><h2>Shaders &#38 Post-process Effects</h2><h3>Shaders</h3><ul><li>A new <code>PER_FACE_LIGHTING</code> flag has been added to the following shaders:<ul><li><code>core/entity.vsh</code><li><code>core/entity.fsh</code></ul><li>When set, cardinal lighting will be computed separately for front- and back-faces<li>As we now require OpenGL 3.3,
