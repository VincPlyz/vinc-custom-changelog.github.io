<p>In this week's snapshot we are adding a Minecraft Server Management Protocol which lets servers query players, tweak game rules, update server settings, and listen for real-time server event notifications. Some server properties have been moved to game rules, allowing them to be updated while the server is running. Additionally, player icons can now be used in chat, the copper golem has received some improvements and we are shipping a bunch of bug fixes.<p>Happy mining!<h1>New Features</h1><ul><li>Added Halychian language support</ul><h1>Changes</h1><h2>Recipes</h2><ul><li>Hanging Signs can now only be crafted using Iron Chains</ul><h2>UI</h2><ul><li>When adding a new server to the server list, the name is no longer prefilled. When left empty the default name is used.<li>More text inputs now support selecting text by mouse dragging</ul><h3>Mouse cursor</h3><ul><li>Some UI components will now change shape of mouse cursor<ul><li>Examples: text inputs, numeric sliders, buttons, scroll bars</ul><li>Additionally, clickable parts of text labels (like links) will now change cursor to the "hand" shape<li>This feature can be disabled with "Allow Cursor Changes" option in "Mouse Settings..." menu</ul><h2>Minor Tweaks to Blocks, Items and Entities</h2><ul><li>Copper Golems hitbox has been tweaked<ul><li>The hitbox is now 0.98 blocks in height to be able to pathfind in one block high spaces</ul><li>Copper Chests created from spawning a Copper Golem now correctly connect to nearby Copper Chests<li>The Copper Golem now uses the Saddle equipment slot for blocks placed atop of its Lightning Rod<ul><li>Its Head equipment slot will now function like a regular head slot, akin to its functionality for humanoid mobs</ul></ul><h1>Technical Changes</h1><ul><li>The Data Pack version is now 85.0<li>The Resource Pack version is now 67.0<li>Added Minecraft Server Management Protocol</ul><h2>Server Properties</h2><p>Several server properties were removed in favour of making them into game rules, allowing these settings to be changed while the server is running.<ul><li>Removed server property <code>allow-nether</code> in favour of new game rule<li>Removed server property <code>spawn-monsters</code> in favour of new game rule<li>Removed server property <code>enable-command-block</code> in favour of new game rule<li>Removed server property <code>pvp</code> in favour of new game rule<li>Added server property <code>management-server-enabled</code> controlling whether the new Minecraft Server Management Protocol is enabled or not<ul><li>Defaults to <code>false</code></ul><li>Added server property <code>management-server-port</code> controlling on what port the Minecraft Server Management Protocol is started<ul><li>Defaults to <code>25585</code></ul><li>Added server property <code>management-server-host</code> controlling with what host the new Minecraft Server Management Protocol is started<ul><li>Defaults to <code>localhost</code></ul><li>Added server property <code>status-heartbeat-interval</code> controlling the interval the management server sends heartbeat notifications to connected clients<ul><li>Defaults to <code>0</code>, meaning it’s disabled</ul></ul><h2>Minecraft Server Management Protocol</h2><p>Added a server management API (JSON-RPC over WebSocket) for dedicated servers<ul><li>The API is disabled by default and can be enabled in the server properties file<ul><li><code>management-server-enabled=true</code><li><code>management-server-host=localhost</code><li><code>management-server-port=25585</code></ul><li>The API is accessible at <code>ws://&#60management-server-host>:&#60management-server-port></code> when enabled<li>Supports querying and updating of server state (players, allowlist, operators, settings, game rules)<li>Sends notifications on state changes (e.g. player joins, game rule updates)<li>Calling <code>{"id":1,"method":"rpc.discover"}</code> returns an API schema containing supported methods and notifications of the currently running server<li>The Data Generator produces an API schema (<code>json-rpc-api-schema.json</code>) in the reports output folder mirroring the contents returned by the <code>rpc.discover</code> method<li>The API adheres to the <a href=https://www.jsonrpc.org/specification rel="noopener noreferrer"target=_blank>JSON-RPC 2.0 specification</a><li>Uses namespaced methods and the reserved namespaces are <code>minecraft</code> (e.g. <code>minecraft:players</code>, <code>minecraft:allowlist/add</code>) and <code>notification</code> (e.g. <code>notification:players/joined</code>)<ul><li>Extensible via custom namespaces for additional methods and events</ul><li>Core method groups: players, allowlist, operators, server (save, stop), server settings, game rules<li>Example method call:<ul><li>Request: <code>{"method":"minecraft:allowlist/add","id":1,"params":[[{"name":"jeb_"}]]}</code><li>Response: <code>{"jsonrpc":"2.0","id":1,"result":[{"id":"853c80ef-3c37-49fd-aa49-938b674adae6","name":"jeb_"}]}</code></ul><li>Example notification:<ul><li><code>{"jsonrpc":"2.0","method":"notification:players/joined","params":[{"id":"853c80ef-3c37-49fd-aa49-938b674adae6","name":"jeb_"}]}</code></ul><li>Example error:<ul><li>Request: <code>{"method": "minecraft:foo/bar","id": 1}</code><li>Response: <code>{"jsonrpc":"2.0","id":1,"result":{"jsonrpc":"2.0","id":1,"error":{"code":-32601,"message":"Method not found","data":"Method not found: minecraft:foo/bar"}}}</code><li>Errors and error codes follow <a href=https://www.jsonrpc.org/specification#error_object rel="noopener noreferrer"target=_blank>JSON-RPC 2.0 error object format</a></ul></ul><h1>Data Pack Version 85.0</h1><ul><li>Renamed <code>chain</code> to <code>iron_chain</code></ul><h2>Game Rules</h2><ul><li>Added <code>pvp</code> game rule controlling whether players can deal damage to other players<ul><li>Defaults to <code>true</code></ul><li>Added <code>allowEnteringNetherUsingPortals</code> game rule controlling if the Nether can be entered through portals<ul><li>Defaults to <code>true</code></ul><li>Added <code>spawnMonsters</code> game rule controlling if monsters should spawn naturally<ul><li>Defaults to <code>true</code></ul><li>Added <code>enableCommandBlocks</code> game rule controlling if Command Blocks should be enabled or not<ul><li>Defaults to <code>true</code></ul></ul><h2>Text rendering</h2><h3><code>object</code> Text Component</h3><ul><li>This component has been updated to support displaying other non-character objects as a part of text<li>New format:<ul><li><code>object</code>: type of object, string, see below for values<li><code>&#60type-specific></code>: additional fields depending on object type</ul><li>Similarly to <code>nbt</code> text component <code>object</code> field can be omitted<ul><li>However, it's recommended to provide it for error checking</ul></ul><h4><code>atlas</code> Object Type</h4><ul><li>Behaves the same way as the <code>object</code> component did previously, but is now a separate object type<li>Uses the same format as before</ul><h4><code>player</code> Object Type</h4><ul><li>Renders player head<li>If partial profile is given, skin is resolved in the same way as <code>minecraft:profile</code> component on player head<li>Format:<ul><li><code>player</code>: player profile<ul><li><code>name</code>: player name, optional string<li><code>id</code>: player uuid, optional UUID<li><code>properties</code>: profile properties, optional map</ul><li><code>hat</code>: controls rendering of a hat layer, boolean, default <code>false</code></ul></ul><h1>Resource Pack
