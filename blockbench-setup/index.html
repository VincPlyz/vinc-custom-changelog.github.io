Blockbench 3D Viewer Setup Guide

1 January 2024·504 words·3 mins

Overview
#

This 3D viewer allows you to embed interactive Blockbench models directly in your Hugo content using a simple shortcode. It supports Minecraft geometry format models with textures, animations, and full mouse/touch controls.

Installation
#

1. Include Three.js Library
#

Add this to your layouts/partials/head.html:

<!-- Three.js Library -->
<script src="https://cdnjs.cloudflare.com/ajax/libs/three.js/r128/three.min.js"></script>

2. Include the Blockbench Viewer Script
#

Add this line to your layouts/partials/head.html or before closing </body>:

<!-- Blockbench 3D Viewer -->
<script src="/js/blockbench-viewer.js"></script>

Performance Tips
#

  1. Optimize Models: Keep polygon count reasonable (under 1000 cubes recommended)
  2. Compress Textures: Use optimized PNG files (512x512 or smaller)
  3. Lazy Loading: The viewer automatically loads when the canvas comes into view
  4. Mobile: The viewer is touch-enabled and responsive

Troubleshooting
#

Model not loading
#

  • Check browser console for errors (F12)
  • Verify the model path is correct relative to /static/
  • Ensure the JSON file is valid Blockbench format

Texture not appearing
#

  • Verify texture file path is correct
  • Check that the texture file exists in /static/
  • Ensure texture dimensions are powers of 2 (512x512, 1024x1024, etc.)

Performance issues
#

  • Reduce model complexity
  • Disable auto-rotate on low-end devices
  • Use smaller textures
